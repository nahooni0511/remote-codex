# Remote Codex UI Design Spec

이 문서는 현재 `/Users/nahooni0511/workspace/remote-codex` 웹 앱의 화면 구조와 시각 규칙을 문서만 보고 비슷하게 재구성할 수 있도록 정리한 디자인 사양서다.

## 1. 디자인 목표

- 전체 인상은 `개발자용 워크스페이스 + 다크 글래스 패널`이다.
- 화면은 정보 밀도가 높지만, 한 번에 너무 많은 보조 정보는 보여주지 않는다.
- 핵심 사용 흐름은 `왼쪽에서 프로젝트/스레드 선택 → 오른쪽에서 메시지 확인/전송`이다.
- Telegram과 Codex가 연결된 도구이지만, Telegram 클라이언트 자체처럼 보이기보다는 `Codex 작업 콘솔`에 가깝게 보인다.

## 2. 시각 언어

### 2.1 컬러 토큰

기본 컬러는 아래 값으로 맞춘다.

- 배경 메인: `#050912`
- 배경 보조: `#0b1220`
- 패널 강한 배경: `rgba(16, 24, 39, 0.96)`
- 일반 패널 배경: `rgba(12, 19, 33, 0.88)`
- 텍스트 메인: `#eef4ff`
- 텍스트 보조: `#8f9cb2`
- 라인/보더: `rgba(148, 163, 184, 0.14)`
- 포인트 컬러: `#4ade80`
- 강한 포인트: `#1f9d55`
- 성공: `#63d49f`
- 경고: `#ffcc66`
- 에러: `#ff7b7b`
- 유저 버블: `rgba(20, 71, 43, 0.96)` 기반 녹색
- 어시스턴트 버블: `rgba(28, 46, 71, 0.92)` 기반 청남색

### 2.2 배경

배경은 단색이 아니라 아래 3겹 조합이다.

- 좌상단 녹색 계열 radial gradient
- 우상단 청색 계열 radial gradient
- 위에서 아래로 어두워지는 linear gradient

결과적으로 `딥 네이비 + 녹색 포인트` 분위기다.

### 2.3 타이포그래피

- 기본 폰트: `SUIT Variable`, `Pretendard Variable`, `IBM Plex Sans KR`, `Segoe UI`, sans-serif 순서
- 기본 글자 크기: `13px`
- 제목은 크지 않다. 실용적인 크기 위주다.
- 대부분의 섹션 제목은 `1.2rem ~ 1.55rem`
- 대메뉴 라벨과 섹션 라벨은 작은 대문자 스타일이다.

### 2.4 패널/모서리/그림자

- 앱 카드 공통 radius: `28px`
- 내부 블록 radius: `26px`
- 입력/버튼 radius: `14px ~ 16px`
- 전체 패널은 흐린 반투명 유리 느낌 + 강한 그림자
- 대표 그림자: `0 28px 80px rgba(0, 0, 0, 0.42)`

## 3. 글로벌 레이아웃

앱은 크게 2단 구조다.

1. 아주 얇은 좌측 대메뉴 rail
2. 그 오른쪽 실제 페이지 콘텐츠 영역

데스크톱 기준 레이아웃:

- 전체 앱: `grid-template-columns: 76px 1fr`
- 간격: `12px`
- 바깥 패딩: `12px`

즉 사용자가 보는 첫인상은 아래 구조다.

```text
[얇은 전역 메뉴] [현재 섹션 화면]
```

## 4. 라우트 구조

현재 화면은 아래 URL 기준으로 구분한다.

- `/setup`
- `/chat`
- `/chat/projects/new`
- `/chat/projects/:projectId`
- `/chat/projects/:projectId/threads/:threadId`
- `/cron-jobs`
- `/config`

핵심은 `Chat` 섹션 안에서만 실제 프로젝트/스레드 작업이 일어난다는 점이다.

## 5. 전역 대메뉴 Rail

### 5.1 위치와 크기

- 화면 맨 왼쪽
- 폭: `76px`
- 전체 높이: `calc(100vh - 24px)`
- sticky 고정

### 5.2 구성

상단:

- 작은 브랜드 블록 `CT`

하단 메뉴:

- `Chat`
- `Cron Job`
- `Config`

### 5.3 스타일

- 버튼은 세로로 쌓인다.
- 각 메뉴 버튼 최소 높이: `68px`
- 활성 메뉴는 녹색 반투명 배경 + 미세한 경계선
- 라벨은 중앙 정렬, 작은 글자

모바일에서는 이 rail이 상단 가로형 메뉴처럼 바뀐다.

## 6. Chat 섹션의 전체 구조

Chat 화면은 다시 2열이다.

- 좌측: 워크스페이스 사이드바
- 우측: 메인 작업 영역

데스크톱 기준:

- 좌측 사이드바 폭: `336px`
- 우측 메인: 나머지 전체
- 두 영역 사이 gap: `14px`

구조는 아래와 같다.

```text
[대메뉴 rail] [프로젝트/스레드 사이드바] [메인 패널]
```

## 7. Chat 좌측 사이드바

### 7.1 상단

- `새 프로젝트` 버튼

### 7.2 본문

- 섹션 라벨: `스레드`
- 프로젝트 목록

### 7.3 프로젝트 항목 구조

프로젝트 1개는 아코디언 카드처럼 보인다.

헤더:

- 접기/펼치기 버튼
- 프로젝트 이름

펼쳤을 때:

- 해당 프로젝트의 스레드 목록
- 스레드는 기본 10개만 노출
- 10개 초과 시 `더보기` 버튼으로 추가 노출

### 7.4 스레드 항목 구조

- 한 줄짜리 타이트한 버튼형 리스트
- 표시 정보는 거의 `스레드 제목`만
- 선택되거나 hover 시 녹색 반투명 강조

### 7.5 사이드바 동작

- 데스크톱에서는 기본 고정
- 모바일에서는 off-canvas 오버레이
- 열렸을 때 반투명 backdrop 표시
- 닫히면 본문만 전체 폭 사용

## 8. Chat 메인 영역 공통 구조

Chat의 우측 메인 영역은 공통적으로 아래 구조를 가진다.

### 8.1 상단 바

왼쪽:

- 사이드바 토글 버튼
- 현재 위치 제목

오른쪽:

- `새로고침` 버튼

### 8.2 콘텐츠 영역

상단바 아래에 실제 페이지 내용이 들어간다.

상태에 따라 다음 중 하나가 보인다.

- 빈 안내 상태
- 새 프로젝트 폼
- 프로젝트 상세
- 스레드 채팅뷰

## 9. 빈 상태

프로젝트도 선택되지 않았고 새 프로젝트 화면도 아닐 때는 아래 한 줄만 보인다.

- `좌측에서 project를 선택하거나 새 project를 만드세요.`

불필요한 그래픽 없이 단순한 안내 텍스트 중심이다.

## 10. Setup 페이지

### 10.1 목적

최초 실행 시 Telegram 사용자 계정 로그인과 bot token 입력을 완료하는 화면이다.

### 10.2 레이아웃

2열 구성:

- 왼쪽: 설명/가이드 영역
- 오른쪽: 실제 입력 폼 영역

비율은 대략 `1.08fr : 1fr`

### 10.3 왼쪽 설명 영역

포함 정보:

- 작은 키커 `Codex x Telegram`
- 큰 제목
- 짧은 설명
- 3개 체크리스트
- `API ID` / `API Hash`를 `my.telegram.org`에서 발급받는다는 안내

성격:

- 소개 랜딩 카드 느낌
- 높이가 충분히 크고 여백이 넓다

### 10.4 오른쪽 폼 영역

1차 폼:

- `Telegram API ID`
- `Telegram API Hash`
- `전화번호`
- `Telegram bot token`
- `로그인 코드 보내기`

조건부 2차 폼:

- `로그인 코드`
- `코드 확인`

조건부 3차 폼:

- `2FA 비밀번호`
- `로그인 완료`

### 10.5 상태 표현

- 에러는 빨간 배너
- 성공은 초록 배너

## 11. 새 프로젝트 페이지

라우트:

- `/chat/projects/new`

### 11.1 목적

새 Codex 프로젝트를 만들고, 동시에 Telegram forum supergroup을 자동 생성하는 화면

### 11.2 정보 구조

폼 항목:

- `그룹 이름`
- `로컬 폴더 경로`
- `폴더 탐색` 버튼

하단 버튼:

- `project 생성`
- `취소`

### 11.3 톤

- 설명 문구는 짧게만 유지
- “그룹 이름과 폴더 경로만 입력하면 자동 생성/연결” 메시지를 보여준다

## 12. 프로젝트 상세 페이지

라우트:

- `/chat/projects/:projectId`

### 12.1 목적

이미 생성된 프로젝트의 기본 정보와 연결 상태를 보여주는 관리 화면

### 12.2 정보 구조

상단 블록:

- 제목 `Project 상세`
- 짧은 설명
- `그룹 이름` 읽기 전용
- `로컬 폴더 경로` 입력
- `폴더 탐색` 버튼
- `Telegram 그룹: {group title}`
- `저장`
- `project 삭제`

하단 블록:

- 제목 `새 thread 만들기`
- `새 thread 생성` 버튼

### 12.3 원칙

- Telegram 연결 관련 값은 “읽기/상태 확인” 중심
- 조작 가능한 핵심은 `폴더 경로`와 `삭제`, `새 thread`

## 13. 스레드 채팅 페이지

라우트:

- `/chat/projects/:projectId/threads/:threadId`

### 13.1 전체 구조

세로 3단 구조:

1. 스레드 헤더
2. 메시지 피드
3. 입력창

### 13.2 헤더 정보

필수 표시:

- 스레드 제목
- 프로젝트 이름
- `model`
- `effort`
- 실행 상태

표시 예:

- `model gpt-5.4 · effort high · idle`

### 13.3 메시지 피드 규칙

메시지 타입은 크게 3개다.

- `system`
- `user`
- `assistant`

#### system

- 일반 텍스트 블록
- 말풍선이 아니라 설명 텍스트처럼 보임
- 색상은 보조 텍스트 색
- 사용 예:
  - Codex 진행 메시지
  - plan 텍스트
  - 첨부 안내

#### user

- 우측 정렬
- 녹색 계열 버블
- 오른쪽 아래 모서리가 살짝 각짐

#### assistant

- 좌측 정렬
- 남청색 계열 버블
- 왼쪽 아래 모서리가 살짝 각짐

### 13.4 메시지 메타

모든 일반 메시지에는 상단 메타 줄이 있다.

- 보낸 사람 이름
- 날짜/시간

### 13.5 첨부 표시

이미지 첨부가 있으면:

- 버블 안 또는 아래에 이미지 표시
- 최대 폭은 `560px`
- 둥근 모서리와 얇은 테두리

파일 첨부가 이미지가 아니면:

- 파일 링크 텍스트로 표시

### 13.6 빈 상태

메시지가 없으면 중앙 정렬 안내 텍스트 표시:

- Telegram topic 또는 입력창에서 첫 메시지를 보내면 Codex 세션이 시작된다는 설명

## 14. 입력창 영역

### 14.1 구조

- textarea 1개
- 우측 하단 정렬의 `보내기` 버튼

### 14.2 입력창 규칙

- 기본 높이: 낮은 한 줄 형태에서 시작
- 자동 높이 확장
- 최대 높이: `220px`
- 그 이상은 내부 스크롤

키보드 규칙:

- `Enter`: 전송
- `Shift + Enter`: 줄바꿈

전송 후:

- 입력값 즉시 비움
- 실패 시 draft 복원

Codex 기본 실행값을 설정하는 관리 페이지

입력 항목:

- `응답 language`
- `기본 model`
- `기본 model_reasoning_effort`

버튼:

- `저장`

### 15.3 폭

- 너무 넓지 않게 `max-width: 720px`

## 16. Cron Job 페이지

라우트:

- `/cron-jobs`

### 16.1 현재 상태

- 단일 패널
- 제목 `Cron Job`
- 설명 `이 영역은 아직 비어 있습니다.`

즉 실제 기능이 아니라 placeholder 성격이다.

## 17. 폴더 브라우저 모달

프로젝트 생성/수정에서 사용하는 팝업 UI다.

### 17.1 동작

- `폴더 탐색` 클릭 시 열림
- 배경은 어두운 backdrop + blur
- 중앙 큰 모달 카드 표시

### 17.2 구조

- 상단 제목/닫기 버튼
- 그 아래 타이트한 디렉토리 트리

### 17.3 트리 규칙

- 노드에는 폴더명만 표시
- 전체 경로는 숨김
- 간격은 매우 타이트
- 현재 선택 경로가 있으면 그 경로까지 기본 expand

## 18. 반응형 규칙

### 18.1 1180px 이하

- Setup 2열이 1열로 바뀜
- `split-grid`도 1열로 바뀜

### 18.2 980px 이하

- 전역 rail이 상단 가로 메뉴로 바뀜
- Chat 사이드바는 고정 컬럼이 아니라 슬라이드 오버레이가 됨
- 메인 워크스페이스는 1열 전체 폭
- 사이드바 열림 시 backdrop 활성화

### 18.3 채팅 영역

모바일에서도 구조는 유지하되,

- 사이드바는 덮개식
- 메인 패널은 최대한 단일 흐름
- 입력창과 메시지 가독성을 우선

## 19. 인터랙션 원칙

- 선택 상태는 `포인트 컬러의 약한 반투명 배경`으로 표시
- destructive 액션은 텍스트로는 명확히 쓰되, 시각적으로 과하게 공격적이지 않다
- 정보 노출은 최소화한다
- 스레드 목록에는 제목 외 메타 정보를 거의 노출하지 않는다
- 헤더에는 현재 컨텍스트를 잃지 않을 최소 정보만 둔다

## 20. 구현 시 반드시 지켜야 할 핵심 포인트

이 문서만 보고 비슷하게 만들 때, 아래는 반드시 유지해야 한다.

- 전체는 `얇은 전역 rail + Chat용 사이드바 + 메인 패널` 구조
- Chat 메인에서 `프로젝트 상세 / 새 프로젝트 / 스레드 채팅뷰`가 같은 프레임 안에서 바뀌는 구조
- 좌측 스레드 탐색은 아코디언 프로젝트 + 10개 단위 더보기
- 채팅은 `system 텍스트`, `user 우측 버블`, `assistant 좌측 버블`
- 다크 글래스 패널, 녹색 포인트, 작은 글자 기반 고밀도 UI
- 모바일에서 사이드바는 오버레이형으로 바뀌어야 함
